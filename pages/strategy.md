- 체크리스트: `pages/encyclopedia-roadmap.md`

## 1) 콘텐츠(무엇을 완벽하게?)

### 1-1. 고정 핵심 페이지(기준점)
- 프로필: `pages/profile.md`
- 타임라인: `pages/timeline.md`
- 작품/캐릭터: `pages/works-characters.md`, `pages/works/*.md`
- 인터뷰/기사: `pages/interviews.md`
- 출연/행사: `pages/appearances.md`
- 화보/사진: `pages/pictorials/*`
- 광고/엠버서더: `pages/endorsements/*`
- 수상: `pages/awards.md`
- 일정: `pages/schedule.md`

### 1-2. 과거 이력 보강 KPI(매일 최소 1개)
- `pages/namu-backlog.md` 항목 중 **최소 1개를 전진**
  - “전진” 정의: **1차 출처 링크 1~3개 확보 → 해당 페이지에 반영 → 체크박스 갱신**

## 2) 출처(어디서?)

### 2-1. 출처 등급
- S(1차): 플랫폼/방송사/제작사/브랜드/매거진 공식, 공식 유튜브/공식 SNS
- A(준1차): 신뢰 매체(원문 인터뷰/보도) + 인물정보(보조)
- B(2차 참고): 나무위키/블로그/커뮤니티(누락 체크용)

원칙:
- B는 **발견/누락 확인용**
- 최종 기록(확정 페이지)은 **S/A 링크로만** 확정

### 2-2. 동적 로딩/레이트리밋 대응
- 동적 사이트는 ‘사이트 내부검색’ 대신:
  - 공식 유튜브 / 공식 인스타 게시물 / 보도자료/기사 원문 링크로 우회

## 3) 자동화(어떻게 매일 같은 품질로?)

### 3-1. 실행/기록 체계
- `news/YYYY-MM-DD.md`
  - `## 실행 상태`: 최신 실행 요약
  - `## 실행 이력`: 모든 실행 기록 누적

### 3-2. 1일 1커밋 권장
- 실행 스크립트: `scripts/run_daily_update.sh`
  - 동시 실행 방지(락)
  - 의미 있는 변경이 없으면 커밋하지 않음

## 4) 품질(QA)

- `scripts/lint_wiki.sh` → `pages/lint-report.md`
- 주간 점검 시에는:
  - 자동 수정 가능한 것만 수정
  - 애매한 건 “수동 확인 필요”로 남김

- 자동 실행 파이프라인: scripts/run_daily_update.sh

## 5) 운영 루틴

- 매일(자동): 09:00 업데이트 → 09:20 푸시
- 매일(검증): 09:25 자동 실행/푸시 성공 여부 확인
- 주 2~3회: 과거 이력 보강 집중(광고/화보/출연 중 테마 선택)
- 매주(월): 린트/빈칸/중복 정리 + 표기 규칙 점검

### 운영 원칙(추천값 적용)
- **수동 실행은 원칙적으로 하지 않습니다.**
  - 09:00/09:05/09:15 자동+재시도로 해결을 우선합니다.
  - 09:25 모니터가 “안됨”으로 보고할 때만, 원인 파악 후 주인님 승인 하에 수동 조치합니다.
- 목적: 커밋/로그 스팸 방지 + 원인 추적 용이 + 자동화 신뢰성 강화
