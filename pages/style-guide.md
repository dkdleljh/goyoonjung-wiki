# 편집 규칙 (Style Guide)

## 1) 문서 톤/언어
- 한국어 존대말, 간결한 문서체로 작성합니다.
- 감상/추측/루머는 배제하고, 사실과 출처 중심으로 기록합니다.

## 2) 날짜/시간 표기
- 기본 날짜: `YYYY-MM-DD`
- 시간대: Asia/Seoul

## 3) 출처 규칙
- 사실(새 정보)은 **반드시 링크**와 함께 기록합니다.
- 출처 우선순위: 공식(소속사/방송사/OTT/시상식/매거진) > 주요 언론 원문 > 2차 정리
- 모든 신규 출처는 `sources/sources.md`에 **접근일**과 함께 추가합니다.

## 4) 신뢰도/검증 상태 태그
각 항목에 아래 중 하나를 붙입니다.
- `상태: 공식확정` (공식 사이트/공식 발표)
- `상태: 공식예고` (공식 예고/티저/라인업)
- `상태: 보도(1차)` (주요 언론 단독/원문)
- `상태: 보도(2차)` (재인용/2차 정리)

## 5) 스포일러 규칙
- 캐릭터/작품 정리는 공식 소개(시놉시스/캐릭터 소개) 범위 우선.
- 스포일러는 반드시 `#### 스포일러` 하위 섹션으로 분리합니다.
- 가능하면 스포일러 강도를 표시합니다: `스포일러: 약|중|강`.
- 가능하면 범위를 표시합니다: 예) `스포일러(EP1-4)`.

## 5-1) 대표작 표기 기준(주관 최소화)
- 원칙적으로 ‘대표작’은 아래 중 하나 이상의 근거가 있을 때만 사용합니다.
  - 공식 플랫폼/방송사가 ‘주연/주요 출연’으로 명시
  - 주요 시상식 수상/노미네이트와 연결
  - 공식 발표된 흥행/화제성 지표(있을 경우)

## 6) 파일 수정 로그(필수)
- 변경이 있으면 `news/YYYY-MM-DD.md`에 다음을 포함합니다.
  - ✅ 오늘 확인한 출처(Watchlist 체크)
  - ✅ 변경 요약(어느 파일/어느 섹션)
  - ✅ 추가/수정한 출처 목록
- 자동화 실행 기록은 `## 실행 상태`(최신 1회) + `## 실행 이력`(누적)으로 남깁니다.

## 6-1) 항목 데이터화(권장)
- 항목을 추가할 때 YAML 메타데이터(프론트매터)를 권장합니다.
- 템플릿: `pages/templates/item-frontmatter.md`

## 7) 스케줄 기록 규칙
- `pages/schedule.md`에는 **공식적으로 공개된 일정**만 기록합니다.
- 일정은 `Upcoming`/`Past`로 분리해 유지합니다.
- 일정 관련 링크도 `id:` + `sources/seen-urls.txt`로 중복 방지합니다.
