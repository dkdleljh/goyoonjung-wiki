# goyoonjung-wiki 현재 구조 분석 및 개선점 식별

## 1. 현재 구조 현황

### 기본 구성
- **메인 페이지**: index.md (77행)
- **핵심 페이지**: 20+ 개별 페이지
- **뉴스 아카이브**: daily 업데이트 시스템
- **자동화**: config.yaml 기반 스크립트

### 강점
1. **체계적인 파일 구조**: pages/ 하위 카테고리별 분류
2. **자동화 시스템**: config.yaml + 스크립트 기반 업데이트
3. **저작권 안전**: 링크 중심의 콘텐츠 정책
4. **실시간 업데이트**: daily news + 자동 스크립트
5. **체계적인 출처 관리**: S/A/B급 출처 분류

### 현재 부족한 정보 필드
#### 1. 프로필 정보 (profile.md)
```markdown
# 현재 필드 (8개 항목)
- 이름, 출생, 직업, 활동, 소속사
- 출생지, 학력 (2차 참고 상태)

# 추가 필요 필드 (Wikipedia 기준)
- Birth Name (본명)
- Height/Weight (신체 정보)
- Other Names (별명, 예명)
- Official Website (공식 사이트)
- Social Media (SNS 계정)
- Agent/Manager (에이전트 정보)
```

#### 2. 수상 정보 (awards.md)
```markdown
# 현재 상태
- 11개 항목, 전부 "교차검증 필요(2차 참고)"
- 공식 링크 없음

# 개선 필요
- 공식 시상식 사이트 링크
- 수상 영상/사진 링크
- 수상식 스피치/인터뷰
```

#### 3. 필모그래피 (filmography.md)
```markdown
# 현재 상태 (초간단 구조)
- 2022년 ~ 2025년 작품 목록
- 기본 정보만 포함

# 추가 필요 정보
- 역할 상세
- 캐릭터명
- 감독/제작사
- 흥행 성적
- 평점/평가
- 시청 정보 (OTT)
```

## 2. 데이터 구조 개선 필요

### 현재 검증 등급 시스템
```
S급(공식) → A급(원문 보도) → B급(2차 참고)
```

**문제점**: 
- 대부분 B급으로 표시되어 신뢰도 저하
- 자동 검증 시스템 부재
- 교차 검증 절차 불명확

### 정보 필드 표준화 부족
1. **날짜 형식**: 일부 혼재
2. **이름 표기**: 한글/영어 통일 필요
3. **작품명**: 원제/부제 표기 규칙
4. **수상명**: 공식 명칭 사용 필요
