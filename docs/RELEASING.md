# Releasing (태그/릴리즈 운영 규칙)

이 저장소는 자동화 러너가 **main 브랜치에 커밋/푸시**한 뒤,
베스트에포트로 **SemVer 태그(vMAJOR.MINOR.PATCH)** 를 생성하고(annotated tag),
가능하면 GitHub Release 도 함께 만듭니다.

## 목표

- 태그/릴리즈는 사람이 봐도 이해 가능한 **단일 규칙**을 가져야 합니다.
- 자동화로 만들어지는 커밋이 많아도, 릴리즈는 **의미 있는 단위**로 남겨야 합니다.

## 태그 규칙 (Canonical)

- 정식 릴리즈 태그: `vMAJOR.MINOR.PATCH` (예: `v1.2.3`)
- 릴리즈 노트: 마지막 릴리즈 태그 이후 커밋 메시지 요약(자동)

> 참고: 과거에 생성된 날짜형/혼재 태그가 존재할 수 있습니다.
> 삭제/재작성(역사 변경)은 리스크가 크므로, **기존 태그는 보존**하고
> 앞으로의 정식 흐름을 `v1.x.y` 라인으로 고정합니다.

## 자동 bump 규칙(요약)

- major: 커밋 메시지에 `BREAKING CHANGE` 또는 `!:` 포함
- minor: 기능/수집기/승격/설정/스키마 성격의 변경
- patch: 버그픽스/문서/운영 개선

## 운영 체크

- main 브랜치 + clean working tree 일 때만 태그를 생성합니다.
- 태그 생성/푸시는 실패해도 daily run 전체를 실패로 만들지 않습니다(베스트에포트).
