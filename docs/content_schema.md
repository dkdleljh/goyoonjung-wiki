# Content Schema (권장 데이터 모델)

> 목적: 항목을 ‘사건 단위’로 기록하고, 시간이 지나며 더 강한 근거(공식/원문)로 자동 승격하기 위한 공통 규격.
> 원칙: 링크-first(저작권 안전), 보수적 승격(확실한 룰만).

## 1) 사건(Event) 단위
- 위키의 기본 단위는 ‘사건(event)’입니다.
  - 예: 예능 출연 1회, 화보 1건, 인터뷰 1건, 브랜드 캠페인 발표 1건.
- 동일 사건에 대한 기사/재인용이 여러 개일 수 있으며, 이 경우 한 사건 블록에 ‘근거 링크’를 누적합니다.

## 2) 상태(Status) 라이프사이클
- 보도(2차) → 보도(1차) → 공식예고 → 공식확정
- 자동화는 ‘승격 후보’까지는 넓게 수집하되,
  - 공식확정 승격은 **공식 도메인/공식 채널** 링크가 확인되는 경우에만 수행합니다.

## 3) id 규칙(중복 방지)
- 기본: `id`는 사건을 대표하는 **canonical URL** 1개로 둡니다.
- URL 정규화 권장:
  - `scripts/normalize_url.py`로 정규화
  - 추가 시 `scripts/add_seen_url.sh`로 seen 등록
- 추적 파라미터(utm 등)는 제거한 URL을 우선합니다.

## 4) 필수 필드(카테고리별)
### Appearances (출연/행사)
- 날짜(YYYY-MM-DD), 구분, 프로그램/행사명, 플랫폼/방송사, 링크(공식/원문), 상태, id

### Interviews/News (인터뷰/기사)
- 날짜, 매체, 제목, 링크, 상태, id

### Endorsements (광고/브랜드)
- 브랜드/회사명, 역할, 발표일/시작일, 링크(공식), 상태, id

## 5) 자동 승격 큐(추천)
- `pages/promotion-queue.md`는 아래를 자동으로 모읍니다.
  - 보도(2차)로 들어온 지 30일 이상인데 공식/원문 승격이 없는 항목
  - 브랜드/작품명이 미확정인 자동 승격 항목
  - i18n 항목 중 allowlist 밖 도메인으로 들어온 후보
